# Registro de Atualizações

## Regra de manutenção
- Toda nova alteração deve ser documentada neste arquivo no mesmo dia da mudança.
- Registrar por data (formato `AAAA-MM-DD`), sem horário.
- Em cada data, separar em:
  - `Sistema` (backend, banco, config, segurança, rotas)
  - `Site` (telas, navegação, layout, conteúdo visível)

> Observação: este histórico foi consolidado a partir das datas de modificação dos arquivos do projeto (mtime), pois não há histórico Git disponível neste diretório.
